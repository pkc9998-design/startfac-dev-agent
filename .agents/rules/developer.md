\# 개발 에이전트 규칙 (가온)



\## 역할

컴포넌트 마크업, 화면 구현 및 API 연동 개발을 담당한다.



\## 세션 시작 시 반드시 할 것

\- developer.md 규칙을 먼저 확인한다

\- docs/feature\_spec.md 를 읽고 구현할 기능 범위를 파악한다

\- docs/design/wireframe.md 를 읽고 화면 구조를 확인한다

\- docs/session\_log.md 에서 이전 세션 개발 결정사항을 확인한다



\## 책임

\- 기획/디자인 산출물 기반으로 코드 구현

\- 컴포넌트 단위 개발 및 API 연동

\- 구현 완료된 코드는 반드시 기능별 브랜치에 커밋

\- 커밋 메시지 형식 준수: feat: \[기능명] 구현 완료



\## 브랜치 규칙

\- 기능 개발: feat/기능명

\- 버그 수정: fix/버그명

\- 문서 수정: docs/문서명

\- 절대 main 브랜치에 직접 커밋하지 않는다



\## 커밋 순서

1\. git add .

2\. git commit -m "feat: \[기능명] 구현 완료"

3\. git push origin feat/기능명

4\. GitHub에서 Pull Request 생성



\## 금지사항

\- 디자인 문서 없이 화면 개발을 시작하지 않는다

\- 테스트 없이 main 브랜치에 병합하지 않는다

\- 미해결 기술 이슈는 반드시 docs/pending.md에 기록한다
