\*\*URL\*\* `/`

\*\*진입 경로\*\* 직접 접속, SNS 공유 링크, QR 코드 스캔



\### 레이아웃 구성



```

┌─────────────────────────────┐

│  \[로고]           \[이용안내] │  ← 상단 GNB

├─────────────────────────────┤

│                             │

│  \[히어로 배너]              │  ← 서비스 핵심 메시지

│  "짐은 여기 두고            │

│   편하게 주차하세요"        │

│                             │

├─────────────────────────────┤

│  이용 방법 요약 (3단계)     │  ← 아이콘 + 텍스트

│  ① 짐 맡기기               │

│  ② 주차하기                │

│  ③ 짐 찾기                 │

├─────────────────────────────┤

│  ⚠ 짐이 3개 이상이신 경우   │  ← 안내 배너

│  예약을 추가로 생성해 주세요│

│  (예약 1건당 최대 2개)      │

├─────────────────────────────┤

│                             │

│  ┌───────────────────────┐  │

│  │    🧳 예약하기        │  │  ← Primary CTA

│  └───────────────────────┘  │

│                             │

│    이미 예약하셨나요?        │

│    \[예약 확인하기 →]         │  ← Secondary 링크

│                             │

└─────────────────────────────┘

```



\### 컴포넌트 상세



| 컴포넌트 | 내용 | 비고 |

|----------|------|------|

| GNB 로고 | BagDrop 서비스명 텍스트 또는 이미지 로고 | 홈으로 이동 |

| GNB 이용안내 | `/guide` 링크 | 텍스트 버튼 |

| 히어로 배너 | 메인 카피 + 서비스 이미지 | 이미지 또는 일러스트 |

| 이용 방법 | 3단계 아이콘 + 짧은 설명 | 가로 스크롤 또는 세로 나열 |

| 짐 초과 안내 배너 | 경고 아이콘 + 안내 문구 | 노란 배경 배너 처리 |

| 예약하기 CTA | Full-width Primary 버튼 | `/reserve/step1` 이동 |

| 예약 확인 링크 | 텍스트 링크 | `/my-reservation` 이동 |

| 위치 | 문구 |

|------|------|

| CTA 버튼 | "예약하기" |

\*\*기능 ID\*\* FO-RES-01, FO-RES-02, FO-RES-03



\### 레이아웃 구성



```

┌─────────────────────────────┐

│  ← 뒤로       1/4 \[====    ]│  ← 진행 표시 (스텝 바)

├─────────────────────────────┤

│  터미널을 선택해 주세요      │  ← 섹션 타이틀

│                             │

│  ┌──────────┐ ┌──────────┐  │

│  │    T1    │ │    T2    │  │  ← 터미널 선택 카드

│  │ 제1여객  │ │ 제2여객  │  │

│  │ 터미널   │ │ 터미널   │  │

│  └──────────┘ └──────────┘  │

├─────────────────────────────┤

│  이용 방향을 선택해 주세요   │  ← 섹션 타이틀

│                             │

│  ┌──────────┐ ┌──────────┐  │

│  │  ✈ 출국  │ │  🏠 귀국 │  │  ← 방향 선택 카드

│  └──────────┘ └──────────┘  │

├─────────────────────────────┤

│  이용 날짜를 선택해 주세요   │  ← 섹션 타이틀

│                             │

│  \[     달력 컴포넌트      ]  │

│                             │

├─────────────────────────────┤

│  ┌───────────────────────┐  │

│  │       다음으로        │  │  ← 다음 버튼 (비활성 → 활성)

└─────────────────────────────┘
```

\### 입력 항목 상세



\#### 터미널 선택 (FO-RES-01)



| 항목 | 내용 |

|------|------|

| 선택지 | T1 (제1여객터미널) / T2 (제2여객터미널) |

| UI 타입 | 카드형 단일 선택 (선택 시 강조 테두리 + 체크 표시) |

| 기본값 | 미선택 |

| 필수 여부 | 필수 |



\#### 이용 방향 선택 (FO-RES-02)



| 항목 | 내용 |

|------|------|

| 선택지 | 출국 / 귀국 |

| UI 타입 | 카드형 단일 선택 |

| 기본값 | 미선택 |

| 필수 여부 | 필수 |

| 보조 설명 | 출국: "공항 떠나기 전 짐 보관" / 귀국: "차 가지러 가기 전 짐 보관" |



\#### 날짜 선택 (FO-RES-03)



| 항목 | 내용 |

|------|------|

| UI 타입 | 인라인 달력 |

| 선택 가능 범위 | 오늘 \~ 오늘로부터 90일 이내 |

| 오늘 이전 날짜 | 선택 불가 (그레이 처리) |

| 필수 여부 | 필수 |



\### 유효성 검증 및 버튼 상태



| 조건 | 다음 버튼 상태 |

|------|--------------|

| 터미널, 방향, 날짜 모두 선택 완료 | ✅ 활성 |

| 하나라도 미선택 | ❌ 비활성 (그레이) |



\### 뒤로가기 동작



| 진입 경로 | 뒤로가기 이동 |

|-----------|--------------|

| 홈에서 진입 | 홈(`/`) |

| 앱 히스토리 없음 | 홈(`/`) |

\## §3. STEP 2 — 짐 정보 입력



\*\*URL\*\* `/reserve/step2`

\*\*기능 ID\*\* FO-RES-04, FO-RES-05, FO-RES-06



\### 레이아웃 구성



```

┌─────────────────────────────┐

│  ← 뒤로       2/4 \[========]│

├─────────────────────────────┤

│  선택 정보 요약 (이전 STEP) │  ← 요약 칩

│  \[T1] \[출국] \[2026.07.15]   │

├─────────────────────────────┤

│  짐 개수를 선택해 주세요    │  ← 섹션 타이틀

│                             │

│  ┌──────────┐ ┌──────────┐  │

│  │          │ │          │  │

│  │   1개    │ │   2개    │  │  ← 개수 선택 카드

│  │          │ │          │  │

│  └──────────┘ └──────────┘  │

│                             │

│  ┌─────────────────────────┐│

│  │ ⚠ 짐이 3개 이상이신    ││  ← 안내 배너 (항상 노출)

│  │ 경우, 예약을 추가로     ││

│  │ 생성해 주세요.          ││

│  │ (예약 1건당 최대 2개)   ││

│  └─────────────────────────┘│

├─────────────────────────────┤

│  요금 안내                  │

│  ┌─────────────────────────┐│

│  │ 예약 1건    5,000원     ││  ← 요금 표시 (고정)

│  │ 짐 개수 무관 동일 요금  ││

│  └─────────────────────────┘│

├─────────────────────────────┤

│  ┌───────────────────────┐  │

│  │       다음으로        │  │

│  └───────────────────────┘  │

└─────────────────────────────┘
```

\### 입력 항목 상세



\#### 짐 개수 선택 (FO-RES-04)



| 항목 | 내용 |

|------|------|

| 선택지 | 1개 / 2개 |

| UI 타입 | 카드형 단일 선택 |

| 기본값 | 미선택 |

| 필수 여부 | 필수 |

| 최대 선택 | 2개 (초과 선택 불가) |



\#### 짐 초과 안내 문구 (FO-RES-05)



| 항목 | 내용 |

|------|------|

| 노출 조건 | 항상 노출 (선택 여부 무관) |

| 위치 | 짐 개수 선택 카드 하단 |

| 문구 | "짐이 3개 이상이신 경우, 예약을 추가로 생성해 주세요. (예약 1건당 최대 2개)" |

| 스타일 | 노란 배경 경고 배너 |



\#### 요금 표시 (FO-RES-06)



| 항목 | 내용 |

|------|------|

| 표시 금액 | 5,000원 (짐 개수 무관 고정) |

| 표시 문구 | "예약 1건 · 5,000원" + "짐 개수와 무관하게 동일한 요금이 적용됩니다." |

| UI | 회색 배경 정보 박스 |



\### 이전 선택값 요약 칩



\- STEP 1에서 선택한 터미널 / 방향 / 날짜를 상단에 작은 칩(chip)으로 표시

\- 수정 불필요 (수정 시 STEP 1로 돌아가기)

\### 레이아웃 구성



```

┌─────────────────────────────┐

│  ← 뒤로       3/4 \[========]│

├─────────────────────────────┤

│  예약자 정보를 입력해 주세요 │

│                             │

│  이름                       │

│  ┌─────────────────────────┐│

│  │ 홍길동                  ││  ← 텍스트 입력

│  └─────────────────────────┘│

│                             │

│  휴대폰 번호                │

│  ┌─────────────────────────┐│

│  │ 010-0000-0000           ││  ← 숫자 입력

│  └─────────────────────────┘│
```

\### 입력 항목 상세



\#### 이름 입력 (FO-RES-07)



| 항목 | 내용 |

|------|------|

| 입력 타입 | `text` |

| 최대 길이 | 20자 |

| 유효성 검증 | 1자 이상 입력 필수 / 공백만 입력 불가 |

| 오류 메시지 | "이름을 입력해 주세요." |

| placeholder | "이름을 입력해 주세요" |



\#### 휴대폰 번호 입력 (FO-RES-08)



| 항목 | 내용 |

|------|------|

| 입력 타입 | `tel` (모바일 숫자 키패드 자동 활성) |

| 형식 | 숫자만 입력, 자동 하이픈 포맷 (010-0000-0000) |

| 유효성 검증 | 01X로 시작하는 11자리 숫자 |

| 오류 메시지 | "올바른 휴대폰 번호를 입력해 주세요." |

| 안내 문구 | "입력하신 번호로 예약 확인 SMS가 발송됩니다." |

| placeholder | "010-0000-0000" |



\### 유효성 검증 및 버튼 상태



| 조건 | 다음 버튼 상태 |

|------|--------------|

| 이름 + 휴대폰 번호 모두 유효 | ✅ 활성 |

| 하나라도 미입력 또는 형식 오류 | ❌ 비활성 |



\### 키보드 처리



\- 이름 입력 완료 후 `다음` 키 → 휴대폰 번호 필드로 포커스 이동

\- 휴대폰 번호 완료 후 `완료` 키 → 키보드 닫힘

\### 컴포넌트 상세



\#### 예약 요약 카드 (FO-RES-09)



| 표시 항목 | 값 |

|----------|-----|

| 터미널 | T1 선택 시 "제1여객터미널" / T2 선택 시 "제2여객터미널" |

| 이용 방향 | "출국" 또는 "귀국" |

| 이용 날짜 | YYYY.MM.DD(요일) 형식 |

| 짐 개수 | "1개" 또는 "2개" |

| 예약자 | 입력한 이름 |

| 연락처 | 입력한 휴대폰 번호 |



\#### 결제 버튼



| 버튼 | 연동 SDK | 동작 |

|------|----------|------|

| 토스페이 (FO-RES-11) | 토스페이먼츠 SDK | 토스페이 결제창 호출 |



\### 결제 처리 흐름



```

\[결제 버튼 클릭]

&#x20;     ↓

\[결제 창 호출 (SDK)]

&#x20;     ↓

&#x20; 성공 ──→ \[결제 승인 API 호출 (서버)]

&#x20;               ↓

&#x20;          \[DB 예약 상태 RESERVED 저장] (FO-RES-12)

&#x20;               ↓

&#x20;          \[SMS 발송 트리거]

&#x20;               ↓

&#x20;          \[/reserve/complete 이동]

&#x20; 실패 ──→ \[결제 실패 안내 모달]

&#x20;               ↓

&#x20;          \[동일 화면 유지, 재시도 가능]

```



\### 결제 실패 처리 (FO-RES-10 / FO-RES-11)



| 케이스 | 처리 방식 |

|--------|----------|

| 사용자 결제창 닫음 (취소) | 결제 화면 유지, 안내 없음 |

| 결제 실패 (잔액 부족 등) | 모달 표시 "결제에 실패했습니다. 다시 시도해 주세요." |

| 네트워크 오류 | 모달 표시 "네트워크 오류가 발생했습니다. 잠시 후 다시 시도해 주세요." |

| 중복 결제 시도 | 서버에서 차단, 모달 표시 "이미 처리 중인 결제가 있습니다." |

\### §8-A. 취소 영역 — 케이스별 분기



\#### Case 1: 취소 가능 상태

\- 조건: 예약 상태 = `RESERVED` AND 이용일 00시 이전



```

├─────────────────────────────┤

│  예약을 취소하시겠어요?     │

│  이용일 전날 자정까지       │

│  100% 환불됩니다.           │

│                             │

│  ┌───────────────────────┐  │

│  │    예약 취소하기      │  │  ← 활성 버튼 (빨간 테두리)

│  └───────────────────────┘  │

└─────────────────────────────┘

```



\#### Case 2: 취소 불가 — 당일 이후

\- 조건: 예약 상태 = `RESERVED` AND 이용일 00시 이후



```

├─────────────────────────────┤

│  ⚠ 이용일 당일은 취소할    │

│  수 없습니다.               │  ← 취소 불가 안내 (텍스트만, 버튼 없음)

└─────────────────────────────┘

```



\#### Case 3: 취소 불가 — 이미 접수됨

\- 조건: 예약 상태 = `STORED`



```

├─────────────────────────────┤

│  짐이 이미 접수된 상태라    │

│  취소할 수 없습니다.        │  ← 안내 텍스트만

└─────────────────────────────┘

```



\#### Case 4: 완료/취소 상태

\- 조건: 예약 상태 = `RETURNED` 또는 `CANCELLED`



```

└─────────────────────────────┘

(취소 영역 없음)

```



\### 취소 확인 모달 (FO-MY-06)

\- 조건: "예약 취소하기" 버튼 클릭 시



```

┌─────────────────────────────┐

│      예약을 취소할까요?     │

│                             │

│  취소 시 결제 금액 전액이   │

│  환불되며, 복구할 수 없습   │

│  니다.                      │

│                             │

│  ┌──────────┐ ┌──────────┐  │

│  │  돌아가기 │ │ 취소하기 │  │

│  └──────────┘ └──────────┘  │

└─────────────────────────────┘

```



| 버튼 | 동작 |

|------|------|

| 돌아가기 | 모달 닫힘, 상세 화면 유지 |

| 취소하기 | 취소 API 호출 → 성공 시 상태 CANCELLED로 갱신 → SMS 발송 → 화면 갱신 |



\### 취소 API 오류 처리



| 케이스 | 처리 |

|--------|------|

| 이미 취소된 예약 | "이미 취소된 예약입니다." 모달 |

| 이용일 경과로 취소 불가 | "취소 가능 시간이 지났습니다." 모달 + 화면 새로고침 |

| 서버 오류 | "취소 처리 중 오류가 발생했습니다. 고객센터로 문의해 주세요." |
