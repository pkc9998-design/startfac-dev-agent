\## 1. 서비스 개요



\### 서비스 한 줄 정의

> \*\*BagDrop\*\* — 인천공항 자가용 이용 여행객을 위한 터미널 입구 짐 임시보관 서비스



\### 핵심 시나리오



| 유형 | 흐름 |

|------|------|

| 출국 | 터미널 입구 도착 → BagDrop 부스에 짐 맡김(QR 발급) → 주차 → 복귀 → QR로 짐 수령 → 탑승 수속 |

| 귀국 | 입국 후 터미널 출구 → BagDrop 부스에 짐 맡김(QR 발급) → 주차장으로 이동 → 차 가져옴 → 복귀 → QR로 짐 수령 → 귀가 |



\### 운영 기본 정보



| 항목 | 내용 |

|------|------|

| 서비스 위치 | 인천공항 T1 / T2 터미널 입구 부스 |

| 운영 형태 | T1 / T2 각각 독립 운영 (공유 없음) |

| 플랫폼 | 모바일웹 (FO: 고객용 / BO: 운영자용) |

| 현장 수기 접수 | 불가 — 온라인 예약 확인 고객만 접수 가능 |

\## 2. 정책 정의 (확정)



\### 2-1. 요금 정책



| 항목 | 내용 |

|------|------|

| 기본 요금 | 예약 1건당 5,000원 고정 |

| 짐 개수 | 예약 1건당 최대 2개 |

| 요금 변동 | 짐 개수 무관 — 1건 = 5,000원 |

| 초과 시 처리 | 짐 3개 이상 → 별도 예약 추가 생성 필요 (FO 안내 문구 노출) |

| 결제 시점 | 예약 시 선결제 (현장 결제 없음) |

| 결제 수단 | 카카오페이, 토스페이 |



> \*\*FO 안내 문구 (확정)\*\*

> "짐이 3개 이상이신 경우, 예약을 추가로 생성해주세요. (예약 1건당 최대 2개)"



\### 2-2. 취소 / 환불 정책



| 시점 | 정책 |

|------|------|

| 이용일 00시 이전 | 100% 환불 가능 (고객 직접 취소) |

| 이용일 00시 이후 | 취소 불가 / 환불 없음 |

| 운영자 강제 취소 | BO에서 처리 가능 (환불 정책은 동일하게 적용) |



> \*\*FO 취소 버튼 노출 조건\*\*: 예약 상태 = `RESERVED` AND 이용일 00시 이전인 경우에만 활성화



\### 2-3. 현장 예약 정책



| 항목 | 내용 |

|------|------|

| 현장 즉석 예약 | 가능 — 현장에서 모바일웹 접속 후 실시간 예약 |

| 현장 수기 접수 | 불가 |

| 현장 현금 결제 | 불가 |

| 예약 없는 현장 보관 | 불가 — 반드시 온라인 예약 완료 후 QR 또는 예약번호 제시 |



\### 2-4. 터미널 운영 정책



| 항목 | 내용 |

|------|------|

| 운영 단위 | T1 / T2 각각 독립 |

| BO 계정 | 터미널별 분리 (T1 운영자 계정 / T2 운영자 계정 별도) |

| 교차 접수 | 불가 — T1 예약은 T1 부스에서만 접수 가능 |

\## 3. 예약 상태 흐름



```

\[RESERVED] 예약완료

&#x20;   │

&#x20;   ├──→ 이용일 00시 이후: 취소 불가

&#x20;   │

&#x20;   ├──→ 이용일 00시 이전 고객 취소 → \[CANCELLED] 취소완료 (100% 환불)

&#x20;   │

&#x20;   └──→ 운영자 QR 스캔 (접수)

&#x20;           ↓

&#x20;       \[STORED] 보관중

&#x20;           │

&#x20;           └──→ 운영자 QR 스캔 (반환)

&#x20;                   ↓

&#x20;               \[RETURNED] 반환완료

```



| 상태 코드 | 명칭 | 설명 |

|-----------|------|------|

| `RESERVED` | 예약완료 | 결제 완료, 부스 미접수 |

| `STORED` | 보관중 | 운영자가 짐 접수 처리 완료 |

| `RETURNED` | 반환완료 | 운영자가 짐 반환 처리 완료 |

| `CANCELLED` | 취소완료 | 고객 또는 운영자에 의해 취소 |

\## 8. 기술 스택 제안



| 레이어 | 기술 | 선정 이유 |

|--------|------|-----------|

| Frontend | Next.js (React) | 모바일웹 SEO·성능 최적화, FO/BO 모노레포 구성 가능 |

| Backend | Next.js API Routes | MVP 단계 FE+BE 통합, 빠른 개발 가능 |

| DB | PostgreSQL (Supabase) | 관계형 + 빠른 셋업, 무료 티어 MVP 운영 가능 |

| 결제 | 카카오페이 + 토스페이 SDK | 국내 모바일 사용자 커버리지 최대 |

| QR 생성 | qrcode.react | 클라이언트 사이드 QR 생성 |

| QR 스캔 | html5-qrcode | 브라우저 카메라 API 기반, 앱 불필요 |

| SMS | 알리고 API (또는 솔라피) | 국내 저렴한 SMS 발송 |

| 인증(BO) | NextAuth.js | 운영자 계정 세션 관리 |

| 배포 | Vercel | Next.js 최적화 배포 |
