\## BO 공통 전제



\### 인증 정책

\- 모든 `/admin/\*` 경로는 로그인 세션이 없으면 `/admin/login`으로 리디렉트

\- 세션 유효 시간: 12시간 (운영 교대 주기 고려)

\- 세션 만료 시 현재 페이지에서 로그인 화면으로 이동, 로그인 완료 후 원래 경로로 복귀



\### 어드민 계정 정책 (단일 계정)

\- 어드민 계정은 1개로 운영 (터미널별 분리 없음)

\- T1/T2 전체 예약 조회, 접수, 반환, 취소 처리 모두 가능

\- 터미널 구분은 권한 제어가 아닌 \*\*필터 및 정보 표시\*\* 수준으로만 활용



\### 공통 GNB (로그인 이후 전 화면)



```

┌─────────────────────────────┐

│ BagDrop 어드민         관리자│  ← 서비스명 + 계정명 (터미널 배지 없음)

└─────────────────────────────┘

```



\### 공통 하단 탭 네비게이션 (로그인 이후 전 화면)



```

┌──────┬──────┬──────┬──────┬──────┐
```

\#### 오늘 현황 카드 (BO-DASH-01, 02, 03)



| 카드 | 표시 값 | 카운트 기준 |

|------|---------|------------|

| 예약 | 오늘 예약 건수 | 이용일 = 오늘 AND 상태 = RESERVED + STORED + RETURNED (취소 제외) AND 터미널 = 선택 필터 |

\### 컴포넌트 상세



\#### QR 스캔 (BO-IN-01)



| 항목 | 내용 |

|------|------|

| 라이브러리 | `html5-qrcode` |

| 활성 조건 | 화면 진입 시 자동으로 카메라 권한 요청 및 스캔 시작 |

| 스캔 성공 | 예약번호 파싱 → 예약 정보 확인 화면으로 즉시 전환 |

| 카메라 권한 거부 | "카메라 접근 권한이 필요합니다. 브라우저 설정에서 허용해 주세요." + 수동 입력 안내 |



\#### 예약번호 수동 입력 (BO-IN-02)



| 항목 | 내용 |

|------|------|

| 형식 | BD-YYYYMMDD-NNNN |

| 자동 변환 | 소문자 입력 시 대문자 자동 변환 |

| 유효성 검증 | 형식 불일치 시 조회 버튼 비활성 |

| 오류 메시지 | "올바른 예약번호 형식이 아닙니다. (예: BD-20260715-0001)" |



\### 오류 처리 (BO-IN-03 \~ 06)



| 케이스 | 오류 메시지 | 동작 |

|--------|------------|------|

| 예약번호 없음 | "해당 예약번호를 찾을 수 없습니다." | 접수 메인 유지 |

| 이미 접수된 예약 (BO-IN-06) | "이미 접수 처리된 예약입니다. (상태: 보관중)" | 접수 메인 유지 |

| 취소된 예약 | "취소된 예약입니다. 접수할 수 없습니다." | 접수 메인 유지 |

| 반환 완료된 예약 | "이미 반환 완료된 예약입니다." | 접수 메인 유지 |

| 이용일 불일치 | "오늘 이용일이 아닌 예약입니다. 예약 날짜를 확인해 주세요." | 접수 메인 유지 |

| 서버 오류 | "처리 중 오류가 발생했습니다. 다시 시도해 주세요." | 접수 메인 유지 |



> 단일 계정이므로 터미널 권한 오류 없음. 단, 접수 확인 화면에서 예약의 터미널 정보를 크게 표시하여 운영자가 물리적으로 올바른 부스인지 스스로 확인할 수 있도록 유도.

> 오류는 모두 모달 또는 하단 토스트로 표시 후 접수 메인 화면으로 자동 복귀. 재스캔 가능 상태 유지.



\### 접수 완료 후 처리 (BO-IN-04)



1\. DB: `RESERVED → STORED` 상태 전환 + 접수 시각 저장

2\. SMS 발송 트리거 (FO-SMS-02: "짐이 접수됐습니다" 고객 알림)

3\. 완료 피드백 화면 표시 (2초 후 자동으로 접수 메인 복귀 OR 버튼 클릭)

\### 기간 탭 상세



| 탭 | 날짜 선택 UI | 집계 범위 |

|----|------------|----------|

| 일간 | 단일 날짜 선택 | 선택일 00:00 \~ 23:59 |

| 주간 | 주 선택 (월\~일) | 해당 주 월요일 \~ 일요일 |

| 월간 | 연월 선택 | 해당 월 1일 \~ 말일 |



\### 터미널 필터



| 선택값 | 집계 범위 |

|--------|----------|

| 전체 | T1 + T2 합산 |

| T1 | 제1여객터미널 예약만 |

| T2 | 제2여객터미널 예약만 |



\### 매출 요약 컴포넌트 (BO-SALE-01)



| 항목 | 계산 방식 |

|------|----------|

| 결제 완료 건수 | 기간 내 RESERVED + STORED + RETURNED 결제 건수 (취소 제외) |

| 취소 / 환불 건수 | 기간 내 CANCELLED 건수 |

| 순매출 | (결제 완료 건수 × 5,000) - (취소 건수 × 5,000) |



\### 결제 내역 목록 (BO-SALE-02)



| 항목 | 내용 |

|------|------|

| 정렬 | 결제 시각 역순 |

| 표시 항목 | 예약번호 / 터미널 배지 / 예약자명 / 결제 수단(토스페이) / 금액 / 결제 시각 |

| 취소 건 표시 | \[환불] 배지 + 금액 앞에 `-` 마이너스 표시 |

| 탭 동작 | 해당 예약 상세(`/admin/reservations/:id`) 이동 |



\### 빈 상태



"선택한 기간에 결제 내역이 없습니다."
